# FIO — глоссарий для русскоязычных текстов

> Цель: убрать англицизмы из чейнджлогов, объявлений команде и пользовательского UI копирайтинга.
> Применять при ревью v3/v4/Top-N announcements и любых docs/ файлов на русском.

## Правила

1. **Финансовые термины** — оставлять английский ТОЛЬКО если он короче или общепринят (VAT, IBAN, FX). Иначе писать по-русски.
2. **Workflow-этапы** — английские названия (Budget Check, Awaiting CEO) ОК, потому что они отражают UI labels. Не переводить в копирайтинге.
3. **Названия фич** — оставлять как в UI (Card Audit, Mark Paid, Confirm for Payment). Иначе ревьюер не найдёт по поиску.

## Терминология FIO-специфичная

| Термин | Что значит | Использование |
|---|---|---|
| Profit Center / PC | Бизнес-стрим (AA, BK, SR, MT, AH, CF, AL) | UI label, не переводить |
| Stream Owner | Владелец стрима | роль; в копирайтинге можно "руководитель направления" |
| Bookkeeper | Бухгалтер | в копирайтинге — "бухгалтер" |
| Holding CEO | CEO холдинга | в копирайтинге — "СЕО холдинга" |
| Card Audit | Закрытие месяца по картам | UI label, не переводить |
| Reconcile | Сопоставление | UI button: "Reconcile" / в тексте: "сопоставить транзакции" |
| Mark Paid | Отметить оплаченным | UI label, не переводить |
| Legal Entity | Юрлицо | в копирайтинге — "юрлицо" |
| Already paid | Уже оплачено | UI label, не переводить |

## Что запрещено

- ❌ Транслитерация без необходимости: «эпрувать», «риджектить», «коммитить» в копирайтинге для нетехнической аудитории (Rita, тестеры)
- ❌ Английский там, где есть короткий русский: «инвойс» вместо «счёт», «вендор» вместо «поставщик» (в копирайтинге)
- ❌ Смешение регистров: «Уже Paid» вместо «уже paid» или «уже оплачено»

## Что разрешено

- ✅ Английские UI labels (Approve, Reject, Submit) в кавычках в русском тексте
- ✅ Имена фич на английском (Card Audit, Confirm for Payment)
- ✅ Профессиональный жаргон в комментариях для команды (deploy, codex, refactor)
